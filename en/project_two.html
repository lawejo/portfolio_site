Creating an online presence for a start-up

In this assignment my group and I worked with a jewelry start-up called Ivaldesmykker. They had an existing homemade wordpress site, an empty instagram account, and a facebook page that had just been initiated. There was no clear coherency in the way Ivaldesmykker expressed themselves online. Our task was to develop a Style & SoMe guide, a video-content to be used in a fashion that would help this particular start-up promote themselves or their products. Finally we had to develop a new website, and this is the part I played a role in.

As I was being the one coding the website in this assignment, the code is what I will focus on while taking you through this project. If you want to check out specific functionalities that I've highlighted underneath here, click on the links to jump to those parts.

Coding mobile first

One of the requirements of this assignment was to code the website with the mobile-first approach, and this entails using media queries with a min-width, since they are active with a screen of a certain minimum width, opposite of the max-width media-queries. On the right is a codepen of the media-queries used in this project.

Some of the advantages of the mobile-first approach is that a majority of users will visit your site on their phone and therefore you can argue that it makes sense to start your design and code there. Another advantage is that the page is loaded with much simpler code and the advanced styling will be added when needed through the media-queries.

See the Pen Media queries by axtecho (@axtecho) on CodePen.

Dynamic data, fetching from a database

This is the semester we were introduced to fetching data and creating a website that is populated with dynamic data. In the codepen on the left is the code I used to fetch JSON-data from our REST API and in the end, an array of objects is sent to the function "handleData".

In the codepen on the left, we have the two functions that take care of the received JSON data, which comes in an array of objects. The first one called handleData serves to take the array and then with the array method .forEach I loop through each object. What then happens to each one is what we see in the function called singlePiece. In it I copy a template from my DOM, with cloneNode(true), which also copies its whole subtree. Then the template is populated with information derived from the object, and in the end it is appended to an empty parent node.

See the Pen Dynamic data by axtecho (@axtecho) on CodePen.
